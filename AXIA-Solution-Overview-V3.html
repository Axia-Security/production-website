Architecture · the pipeline

One pipeline, five stages.
Every capability lives in a stage.

Ingest across every channel → judge at the gate (millisecond Gatekeeper) → render the verdict (near real-time Judge) → orchestrate the action → learn from every close. Each stage is an independent service with failure isolation, audit logging, and explainability at every boundary.

axiasecurity.iostage 1ingest
Stage 2 · Understand · the millisecond Gatekeeper

A local classifier judges WHO × WHAT × to WHOM —
in milliseconds, with no egress.

A ModernBERT-class primary classifier plus complementary models under confidence-weighted soft voting, on a five-level severity taxonomy (Critical / High / Medium / Low / Non-sensitive). It runs locally in your environment, makes no LLM calls, and emits no content — it classifies every event in milliseconds and gates only what's worth judging. The file and user priors sharpen it; neither is required.

THE REASONING

A verdict, not a score

The Judge — an LLM — renders what the content is, what's sensitive about it, the apparent intent, and which behavioral and organizational signals raise or lower severity.

A PER-VERDICT FIELD

next-best-action

Every verdict carries the single recommended move, attached to the verdict object itself — later acted on by a human, or executed automatically where you've consented (Settings, per action and severity band).

Events the Gatekeeper flags go to the near-real-time Judge. AXIA never sees content under any mode — the Judge runs against an LLM endpoint provisioned per engagement (next slide). The verdict object IS the audit artifact: the same structure that drives the action is the one your reviewer reads.

The engine · two tiers, three endpoint modes

A local millisecond Gatekeeper.
A near-real-time Judge — on your model.

MODE (a) · CUSTOMER CLOUD

Private Cloud default

Azure OpenAI, Amazon Bedrock, or Google Vertex AI in your own account. The default for Microsoft / AWS / GCP estates.

MODE (b) · CUSTOMER-INTERNAL

On-Premises default

An open-weights model (Llama 3.3, Mistral, Qwen) hosted by your IT on your GPUs. The default for finance, government, defence.

MODE (c) · AXIA-PROVIDED FALLBACK

When neither is ready

Llama 3.3 70B via vLLM on customer-provisioned infrastructure, with an open-weights swap path.

Tier 1 · Gatekeeper — local, millisecond, no egress Tier 2 · Judge — near real-time, never a numeric SLA

Under all three modes, AXIA never sees content — the endpoint is always in, or provisioned by, the customer. Mode (c) runs on customer-provisioned infrastructure, not AXIA infrastructure. This is what keeps the perimeter commitment true.
